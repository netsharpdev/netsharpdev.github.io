How should art-monkey appear in your portfolio?

Two structural approaches — both capture the same work, different framing

A

One combined entry — "Art Monkey CMS System"

Single portfolio item covering the full system: the public Next.js website (artmonkey.pl) + the .NET Azure Functions CMS API + the React admin UI. One description, two screenshots (website + admin panel).

Best when the value is the end-to-end system, not the parts individually.

B

Two separate entries — website + CMS

Entry 1: artmonkey-stage — the creative studio portfolio website (Next.js, Azure CDN, animations).
Entry 2: art-monkey-cms — the headless CMS backend + admin panel (.NET 10 Azure Functions, React, Tailwind).

Best when you want to highlight different skills (frontend vs backend/infra) separately.
