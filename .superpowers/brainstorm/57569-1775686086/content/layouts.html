Which page layout feels right?

All use a problem-first structure — the difference is rhythm and depth

A

Landing Page Style

High-impact hero with bold problem statement → pain point bullets → 4 service cards in a grid → trust signals (courses, experience) → single CTA. Punchy, scannable, conversion-focused.

► Hero: "Is your business ready for AI?" + bold subhead
► Pain points: 3 bullet problems SMEs face
► Services: 4-card grid (Audit / Workshop / Implementation / Advisory)
► Trust: credentials, courses, years of experience
► CTA: "Let's talk" → Contact
B

Narrative Scroll

Tells a story: "Here's the problem SMEs face" → "Why now?" → "My approach" → expanded service sections → credentials → CTA. More persuasive, editorial feel, works well for a longer read.

► Hook: "Your competitors are already using AI"
► The problem: paragraph-form, relatable SME context
► Why me: approach + philosophy
► Each service: full section with details
► Social proof: credentials, courses
► CTA: "Start the conversation" → Contact
C

Problem → Service Match

Each problem is paired with its solution service. "Struggling with X? → AI Audit. Team not using AI? → Workshop." Highly direct, easy to self-qualify, great for SME decision-makers who know their problem.

► Intro: "I help SMEs solve these AI challenges"
► 4 paired sections: Problem ↔ Service
► Each pair: symptom, what you get, outcome
► Credentials: compact trust section
► CTA: "Which challenge is yours?" → Contact
